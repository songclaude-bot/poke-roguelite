## 1. 게임 개요

### 한 줄 요약
"포켓몬을 모으고 키우며 랜덤 던전을 탐험하는 턴제 로그라이트"

### 핵심 차별점
- **Shattered Pixel Dungeon**의 간결한 모바일 UX
- **PMD**의 포켓몬 세계관 + 동료 모집
- **Hades**의 메타 진행 시스템

### 플랫폼
- 모바일 웹 (세로 모드, PWA)
- Phaser 3 렌더링 + rot.js 던전 생성

### 한 판 플레이 시간
- 짧은 던전: 5~10분 (5층)
- 보통 던전: 15~20분 (10~15층)
- 심층 던전: 30~45분 (25~50층)

## 3. 턴 시스템 & 이동

### 기본 규칙
- **1턴 = 1행동**: 이동 1칸, 기술 사용, 아이템 사용, 대기 중 택1
- 내 턴 → 동료 AI 턴 → 적 턴 (동시 해결이 아닌 순차)
- 속도(Speed)에 따라 1턴에 2회 행동 가능 (Speed > 100)

### 이동
- 8방향 이동 (상하좌우 + 대각선)
- 조작: 이동할 타일 탭 or 스와이프 방향
- **자동 이동**: 긴 복도에서 탭하면 적 만날 때까지 자동 전진

### 타일 종류 (원본 DTEF 시스템 그대로 활용)
| 타일 | TerrainType | 설명 |
|------|-------------|------|
| 바닥 | GROUND | 누구나 이동 가능 |
| 벽 | WALL | 이동 불가. 고스트 타입은 통과 가능 (턴당 배고픔 5배 소모) |
| 물 | WATER | 물/비행 타입만 이동 가능. 나머지는 못 지나감 |

### 시야 (FOV)
- rot.js의 FOV 알고리즘 사용
- 방 안: 방 전체 보임
- 복도: 전방 2칸만 보임
- 어둠 속성 던전: 전방 1칸만 보임

## 4. 전투 시스템

### 기본 공격
- 인접한 적에게 자동 기본 공격 (해당 방향 탭)
- 데미지 = ATK × 타입 보정 - DEF × 0.05

### 기술 시스템 (최대 4개 장착)
- PP 소모 방식: 각 기술에 PP가 있고, 사용하면 감소
- PP 회복: 이동 시 1턴당 1PP 자연 회복 / 맥스업씨 등 아이템
- 기술 범위 패턴 (원본 536개 기술에서 선별):

| 범위 유형 | 설명 | 예시 |
|-----------|------|------|
| 정면 1칸 | 기본 근접 | 할퀴기, 몸통박치기 |
| 정면 직선 | 원거리 관통 | 냉동빔, 10만볼트 |
| 주변 8칸 | 범위 공격 | 지진, 파도타기 |
| 방 전체 | 광역 | 열풍, 붐볼 |
| 자신 | 버프/회복 | 칼춤, 회복 |
| 정면 2칸 | 중거리 | 불꽃세례 |

### 타입 상성 (PMD 방식 — 압축된 배율)
| 상성 | 배율 | 이유 |
|------|------|------|
| 효과 굉장 | ×1.3 | 원작 ×2.0은 원킬 너무 쉬움 |
| 보통 | ×1.0 | - |
| 효과 별로 | ×0.8 | 약해도 무의미하진 않음 |
| 면역 | ×0.5 | 완전 면역은 없음 (PMD 전통) |

### 상태이상 (원본 30종에서 핵심 선별)
**디버프** (10종):
- 화상: 매 턴 HP 감소, ATK 저하
- 독: 매 턴 HP 감소 (맹독은 누적)
- 마비: 50% 확률로 행동 불가
- 빙결: 행동 불가, 피격 시 해제
- 수면: 행동 불가, 피격 시 해제, 매 턴 HP 회복
- 혼란: 이동/공격 방향 랜덤
- 매혹: 적을 공격 불가, 피격 시 해제
- 풀죽음: 기술 사용 불가 (기본공격만)
- 속박: 이동 불가
- 저주: 매 턴 HP 대폭 감소

**버프** (5종):
- 칼춤: ATK 상승
- 방어: 1턴 데미지 무효
- 배리어: DEF/SPE_DEF 상승
- 가속: 1턴에 2회 행동
- 부활: 1회 전투불능 방지

## 5. 포켓몬 1,200종 활용 구조

> **"1마리만 쓰는 게임"이 아니라 "1,200마리가 세계를 이루는 게임"**

### 5-1. 플레이어 팀 (3마리)

```
┌──────────────────────────────┐
│  리더 (직접 조작)             │
│  + 동료1 (AI, 전술 설정 가능) │
│  + 동료2 (AI, 전술 설정 가능) │
└──────────────────────────────┘
```

- **리더**: 플레이어가 직접 조작하는 포켓몬. 이동/기술/아이템 모두 직접 선택
- **동료**: AI 자동 행동. 전술 설정 가능:
  - 공격적: 적 보이면 돌진
  - 신중하게: 리더 근처에서만 행동
  - 따라오기: 전투 안 함, 리더 뒤에 붙어다님
  - 거리 유지: 원거리 기술만 사용

### 5-2. 포켓몬 역할 분배

| 역할 | 포켓몬 수 (약) | 설명 |
|------|:---:|------|
| 스타터 (초기 해금) | ~25종 | 1세대 스타터 라인 + 인기 포켓몬. 타입별 2~3종 |
| 동료 모집 가능 | ~400종 | 던전에서 적으로 등장 → 격파 시 모집 확률 |
| 일반 적 전용 | ~500종 | 던전 적으로만 등장. 진화 전 형태 등 |
| 보스 | ~50종 | 전설/환상 포켓몬. 던전 최하층 보스 |
| 특수 해금 | ~100종 | 도전과제, 이벤트, 특수 조건으로 해금 |
| 진화체 | ~300종 | 모집한 포켓몬을 진화시켜 획득 |

### 5-3. 동료 모집 시스템

- 적 포켓몬을 **리더가 직접 마지막 일격**으로 쓰러뜨리면 모집 판정
- 기본 모집률: 레어도에 따라 다름

| 레어도 | 기본 모집률 | 비고 |
|--------|:---------:|------|
| COMMON | 15% | 자주 등장, 모집 쉬움 |
| UNCOMMON | 10% | - |
| RARE | 5% | 약간 어려움 |
| EPIC | 2% | 꽤 어려움 |
| ULTRA | 0.5% | 매우 어려움 |
| UNIQUE | 모집 불가 | 스타터/특수 해금 전용 |
| LEGENDARY | 보스 클리어 시 | 해당 던전 첫 클리어 보상 |

- 모집률 보정: 리더 레벨 (+0.5%/Lv), 친화의리본 아이템 (+10%), 시설 업그레이드 (+5%)
- 모집 성공 시 **해당 런에서 바로 팀에 합류** (4번째 슬롯으로 임시 합류, 마을 귀환 시 정식 등록)
- 실패해도 **도감에 "목격" 등록** → 도감 수집 동기

### 5-4. 진화 시스템

| 진화 유형 | 조건 | 예시 |
|-----------|------|------|
| 레벨 진화 | 특정 레벨 도달 | 파이리 Lv16 → 리자드 |
| 아이템 진화 | 진화석 사용 | 피카츄 + 천둥의돌 → 라이츄 |
| 조건 진화 | 특수 조건 충족 | 이브이 + 친밀도MAX → 에브이 |
| 스택 진화 | 같은 종 N마리 합성 | 3성 시스템 (오토체스 방식 선택적) |

- 진화는 **마을의 진화 시설**에서만 가능 (던전 내 불가)
- 진화 시 스프라이트/스탯/기술풀 변경

### 5-5. 적 출현 테이블

각 던전의 3가지 시너지(타입)에 맞는 포켓몬이 출현:

```
예) 해변동굴 (AQUATIC, GROUND, FOSSIL)
  → 코다크, 크랩, 디그다, 투구, 프테라 등

예) 크리스탈동굴 (PSYCHIC, SOUND, FAIRY)
  → 유니란, 리그레, 삐삐, 치렁, 노래기 등

예) 마그마동굴 (FIRE, GROUND, ROCK)
  → 마그카르고, 마그비, 꼬마돌, 디그다, 누오 등
```

- 원본 `RegionDetails`의 던전별 3시너지 매핑을 그대로 활용
- 층이 깊어질수록 높은 레어도 포켓몬 등장
- 10% 확률로 "해당 던전 테마 외" 포켓몬 등장 (변화 부여)

## 6. 아이템 시스템

### 인벤토리
- **최대 20칸** (가방 업그레이드로 최대 32칸)
- 팀 공용 인벤토리 (리더가 관리)

### 아이템 카테고리 (원본 190+에서 선별)

| 카테고리 | 종류 수 | 던전 내 효과 |
|----------|:------:|------------|
| **식량** | 5 | 사과(배고픔+50), 큰사과(+100), 구미(타입별 경험치) |
| **열매** | 15 | 오렌열매(HP회복), 리샘열매(상태이상회복), 유루열매(혼란해제) 등 |
| **씨앗** | 10 | 수면씨(적 수면), 폭발씨(범위데미지), 부활씨(전투불능방지) 등 |
| **오브** | 10 | 텔레포트오브(랜덤이동), 탈출오브(던전탈출), 폭파오브(방전체공격) |
| **장비** | 20 | 스카프/밴드류. 1인 1개 장착. 스탯보정/패시브 |
| **진화석** | 10 | 물의돌, 불꽃의돌, 천둥의돌 등. 마을에서 진화용 |
| **TM** | 30 | 기술 가르침. 소모품 |
| **투척** | 5 | 돌멩이(고정데미지), 가시(3데미지) 등 |
| **스테이터스** | 5 | 맥스업씨(PP전회복), 단련씨(스탯영구+1) |

### 배고픔 시스템
- 배고픔 게이지: 0~100
- 이동 1턴당 -0.1 (약 1,000턴 탐색 가능)
- 고스트 벽 통과: 턴당 -0.5
- 배고픔 0: 매 턴 HP -1
- 사과 드랍률로 난이도 조절

## 7. 던전 설계

### 7-1. 절차적 던전 생성

**기술 스택**: rot.js Digger/Uniform 알고리즘 → TerrainType[50][32] 배열 → DTEF 오토타일링 → Phaser 3 타일맵

```
rot.js 던전 생성 (방+복도 구조)
    ↓
TerrainType[][] 배열 (WALL/GROUND/WATER)
    ↓
Masker.mask8bits() (47-state 오토타일링)
    ↓
Tileset.getTileId() (타일 ID 계산)
    ↓
Tiled JSON export → Phaser 3 렌더링
```

- 맵 크기: **32×24 타일** (모바일 최적화, 원본 50×32보다 작게)
- 타일 크기: 24×24px, 2배 렌더링 = 48×48px
- 한 화면에 보이는 범위: ~7×10 타일 (카메라 추종)

### 7-2. 층 구성 요소

| 요소 | 설명 |
|------|------|
| 방 (Room) | 5~15개. 크기 4×4 ~ 8×8. 적/아이템 배치 |
| 복도 (Corridor) | 방과 방을 연결. 폭 1~2칸 |
| 계단 | 층당 1개. 가장 먼 방에 배치 |
| 아이템 | 방 바닥에 랜덤 배치 (3~8개/층) |
| 적 | 방에 랜덤 스폰 (5~15마리/층, 층 깊이에 비례) |
| 함정 | 보이지 않는 타일. 밟으면 발동 (수면, 독, 워프 등) |
| 몬스터하우스 | 10% 확률. 방 진입 시 적 8~16마리 동시 출현 |
| 상점 | 20% 확률. 카클레온 상점 (아이템 구매/판매) |

### 7-3. 던전 목록 (98종 타일셋 활용)

| 천둥파동동굴 | ThunderwaveCave | 전기/강철/바위 | 레쿠쟈 |

**중반 던전** (10~15층, 초반 클리어 후 해금):

| 크리스탈동굴 | CrystalCave1 | 에스퍼/소리/요정 | 크레세리아 |

### 7-4. 날씨 시스템

- 5층마다 날씨 변경 (해당 던전 시너지와 연동)
- 날씨 효과:

| 날씨 | 시각 효과 | 게임 효과 |
|------|----------|----------|
| 맑음 (ZENITH) | 밝은 조명 | 풀 타입 기술 +20% |
| 비 (RAIN) | 빗방울 파티클 | 물 타입 +20%, 불 -20% |
| 모래바람 (SANDSTORM) | 모래 파티클 | 바위/땅/강철 DEF+2, 나머지 매턴 HP-1 |
| 눈 (SNOW) | 눈 파티클 | 얼음 타입 +20%, 빙결 확률 증가 |
| 안개 (MISTY) | 안개 오버레이 | 시야 -1, 요정 타입 +20% |
| 밤 (NIGHT) | 어두운 틴트 | 어둠/고스트 +20% |

### 시설 업그레이드 (메타 진행)

| 시설 | 레벨 | 효과 | 비용 |
|------|:---:|------|------|
| 창고 | 1→5 | 보관 +20칸씩 (20→100) | 누적 골드 |
| 가방 | 1→3 | 던전 내 인벤토리 +4칸씩 (20→32) | 누적 골드 |
| 훈련장 | 1→5 | 골드로 경험치 구매 효율 증가 | 누적 골드 |
| 카클레온상점 | 1→5 | 상점 라인업 확장, 할인율 증가 | 누적 골드 |
| 모집캠프 | 1→5 | 기본 모집률 +2%씩 (→+10%) | 보스 코어 |
| 진화의동굴 | 1→3 | 진화 가능 종 확장 (조건진화 해금) | 보스 코어 |
| 기술교실 | 1→3 | TM 외 기술 리마인더 해금 | 보스 코어 |
| 쥬크박스 | - | 클리어한 던전 BGM 재생 | 자동 해금 |

### 재화

| 재화 | 획득 | 용도 |
|------|------|------|
| **골드** (포켓) | 던전 내 드랍, 아이템 판매 | 상점 구매, 시설 업그레이드, 훈련 |
| **보스 코어** | 보스 첫 클리어 시 1개 | 고급 시설 업그레이드 |
| **별의파편** | 심층 던전, 도전과제 | 특수 해금 (희귀 스타터, 코스튬) |

## 9. 메타 진행 설계

### 영구 진행 요소 (죽어도 유지)

| 요소 | 설명 |
|------|------|
| 모집한 포켓몬 | 한 번 모집하면 영구 보유. 죽어도 사라지지 않음 |
| 도감 | 목격/포획/진화 기록 영구 저장 |
| 시설 레벨 | 업그레이드한 시설은 영구 |
| 재화 (골드) | 마을 귀환 시 전액 보존 (사망 시 50% 보존) |
| 해금 던전 | 한 번 해금하면 영구 |
| 보스 첫 클리어 | 전설 포켓몬 + 보스 코어 획득 |

### 런 내 진행 요소 (죽으면 리셋)

| 요소 | 설명 |
|------|------|
| 레벨 | 던전 내에서 올린 레벨은 유지 (사망 시 -5레벨 페널티) |
| 아이템 | 사망 시 던전 내 아이템 전부 유실 (보험 아이템 제외) |
| 층 진행 | 던전은 처음부터 다시 |

### 해금 단계

```
게임 시작
  → 스타터 5종 중 택1 (피카츄, 이상해씨, 파이리, 꼬부기, 이브이)
  → 해변동굴 해금

해변동굴 클리어
  → 작은초원 + 천둥파동동굴 해금
  → 스타터 5종 추가 해금

초반 3던전 모두 클리어
  → 중반 던전 3종 해금
  → 진화의동굴 해금
  → 스타터 10종 추가 해금

중반 클리어
  → 후반 던전 해금
  → 심층 던전 일부 해금
  → "어려운 모드" 해금 (적 레벨 +10, 보상 +50%)

후반 클리어
  → 전 심층 던전 해금
  → 색이 다른 포켓몬 출현 활성화
  → 특수 도전 던전 해금
```

## 11. 기술적 아키텍처

### 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 엔진 | Phaser 3 | 렌더링, 입력, 오디오, 타일맵, 씬 |
| 던전 생성 | rot.js | Digger/Uniform 알고리즘, FOV, A* |
| 언어 | TypeScript | 타입 안전성 |
| 오토타일링 | DTEF Masker (원본 코드) | 47-state 비트마스크 |
| 애니메이션 타일 | phaser-animated-tiles | 물/용암 타일 애니메이션 |
| 저장 | localStorage + IndexedDB | 메타 진행 저장 |
| 배포 | GitHub Pages (PWA) | 정적 호스팅 |
| 빌드 | Vite 또는 esbuild | 빌드/번들링 |

### 저장 구조

```typescript
// localStorage (가벼운 데이터)
interface SaveData {
  version: string
  player: {
    gold: number
    bossCore: number
    starShard: number
  }
  pokedex: Record<PkmId, {
    seen: boolean
    caught: boolean
    evolved: boolean
  }>
  team: {
    leader: PkmId
    members: PkmId[]  // 보유 포켓몬 ID 목록
  }
  facilities: Record<FacilityId, number>  // 시설 레벨
  unlockedDungeons: DungeonId[]
  settings: GameSettings
}

// IndexedDB (큰 데이터)
interface PokemonStorage {
  id: string
  species: PkmId
  level: number
  exp: number
  moves: AbilityId[]
  heldItem: ItemId | null
  stats: PokemonStats
  isShiny: boolean
  nickname?: string
}
```

예상 용량: ~50KB (localStorage 충분). 포켓몬 수가 수백 마리가 되면 IndexedDB로 이관.

### 성능 최적화

| 과제 | 해결책 |
|------|--------|
| 스프라이트 로딩 (1,200종) | 현재 층 출현 포켓몬만 동적 로드 (Lazy Loading) |
| 타일맵 렌더링 | Phaser 3 Tilemap 컬링 (화면 밖 렌더링 안 함) |
| 적 포켓몬 다수 | 오브젝트 풀링 (생성/파괴 대신 재사용) |
| 이펙트 | 스프라이트 시트 공유, 파티클 풀링 |
| 모바일 메모리 | 층 이동 시 이전 층 에셋 해제 |
| FOV 계산 | rot.js PreciseShadowcasting (최적화됨) |

### 에셋 로딩 전략

```
게임 시작 시 로드:
  - UI 에셋 (아이콘, 폰트, HUD)
  - 타입 아이콘 31종
  - 공통 이펙트 (피격, 상태이상)

던전 진입 시 로드:
  - 해당 타일셋 PNG/JSON (1종)
  - 날씨 파티클 텍스처
  - BGM 1트랙

층 진입 시 로드:
  - 해당 층 출현 포켓몬 스프라이트 (~10종)
  - 해당 층 기술 이펙트

팀 편성 시 로드:
  - 리더 + 동료 스프라이트/초상화
```
